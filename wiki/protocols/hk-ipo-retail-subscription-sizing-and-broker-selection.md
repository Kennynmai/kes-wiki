# 港股 IPO 散户申购金额与券商选择协议

## Purpose

用于判断 Kenny 参与港股打新时应该申购多少、是否使用孖展、如何观察认购热度、以及如何选择券商。

这不是买卖建议。港股 IPO 的收益高度依赖市场情绪、发行结构、暗盘流动性、融资成本和最终分配结果；每次申购前都需要重新核对招股书、券商页面和配售结果公告。

## Core idea

每只新股理论上都有一个“相对最优申购档位”，但只能估算，不能事前精确知道。

申购金额越大，中签概率通常会上升，但不是线性上升；融资利息、认购手续费、暗盘/卖出成本和机会成本却更接近线性增加。热门股尤其容易出现“多加很多杠杆，只是从几乎不中变成低概率中一手”的情况。

简化公式：

```text
期望收益 =
  预计中签股数 * 预计每股首日利润
  - 冻资 / 融资成本
  - 认购、交易、平台和暗盘费用
  - 破发、低开、无法及时卖出的风险成本
```

## Sizing ladder

优先比较三个档位，而不是随意填金额：

| 档位 | 作用 | 适用情形 | 主要风险 |
|---|---|---|---|
| 一手 | 低成本买彩票 | 极热、货源少、中签率极低，或只是观察市场 | 大概率不中 |
| 甲尾 | 散户组内的效率观察点 | 甲尾中签概率明显高于一手，且成本可接受 | 融资成本上升，仍可能只中一手 |
| 乙头 | 用资金换更高获配概率 | 判断首日涨幅大，且乙头比甲尾明显改善中签或获配 | 杠杆和利息放大，破发损失更明显 |

如果甲尾和乙头的预估中签率差距不大，不应为了“看起来更积极”冲乙头。

## Decision rules

- 先判断发行机制：机制 B 无强制回拨时，即使超购极高，公开发售货源也可能不增加，散户申购效率会下降。
- 先看公开货源：公开发售手数越少，热门股的一手中签率越容易被压到极低。
- 再看孖展热度：孖展倍数越高，不等于越值得申购；它同时代表市场认可和拥挤度上升。
- 比较边际中签：如果从一手到甲尾的概率改善明显，可以考虑甲尾；如果从甲尾到乙头改善有限，应停止加码。
- 只在预期暗盘/首日涨幅足以覆盖融资成本时使用杠杆。
- 对有诉讼、信息披露争议、亏损扩大、无绿鞋、无基石或行业景气弱的新股，降低申购档位或放弃。
- 对超热门股，不要把“高倍孖展”误读成“高确定性收益”；高倍孖展常常意味着中签效率变差。

## Margin signal workflow

观察孖展认购时按时间层级看：

1. 招股首日：是否快速超购，判断基础热度。
2. 倒数第二日：孖展是否持续加速，判断资金是否仍在追加。
3. 截止前半日：是否爆炸式上升，判断拥挤程度。
4. 公布配售结果：用真实公开认购倍数、一手中签率、稳中一手手数校准前期估算。
5. 暗盘：验证市场愿意给出的实际溢价。

常用观察源：

- AASTOCKS 新股频道：招股资料、暗盘、超购倍数和上市表现。
- TradeSmart IPO Tracker：孖展总额、超购倍数和中签率估算。
- 券商 App：富途、老虎、华盛、辉立、长桥等的新股页面，观察融资额度、融资利率、截止时间和暗盘支持。
- 港交所 / 披露易配售结果公告：最终确认公开认购倍数、分配基准、一手中签率和回拨情况。

## Broker selection criteria

券商选择不是单点最优，而是按用途拆分：

| 用途 | 更重要的指标 | 选择倾向 |
|---|---|---|
| 现金一手 | 现金认购费、入金便利、App 稳定 | 低或免现金认购费的平台 |
| 融资打新 | 融资利率、杠杆倍数、融资额度、截止时间 | 低息且额度充足的平台 |
| 暗盘卖出 | 暗盘流动性、报价深度、交易稳定性 | 富途、辉立等暗盘流动性强的平台优先 |
| 高频参与 | 总费用、操作效率、提醒、历史记录 | 费用透明且流程顺的平台 |
| 资金安全 | 监管牌照、结算参与者身份、出入金能力 | 大平台或本地老牌券商优先 |

## Practical default

- 只想简单参与：用一个主券商，优先选择暗盘流动性强、App 稳定、出入金顺的平台。
- 主要做一手低成本：选现金认购费用低的平台，避免为了开户奖励频繁切换。
- 主要做融资：每只新股逐项比较融资利率、杠杆、额度和截止时间；不要默认某券商永远最便宜。
- 很重视暗盘退出：优先选择暗盘成交活跃的平台，因为能否及时卖出比少几十港元手续费更重要。

## Red flags

- 只看孖展倍数，不看公开货源和发行机制。
- 用高杠杆申购所有热门股，不区分行业、估值和发行结构。
- 忽略延期上市、取消上市时手续费和融资利息可能不退。
- 把券商促销利率当成长期条件。
- 用多个同名账户重复申购同一只新股，导致申请无效风险。
