# 跨境电商运营绩效方案设计框架

## Trigger / scope

当 KES 需要为跨境电商运营团队制定、调整或复盘绩效方案时，使用本协议。

适用范围：
- Amazon 成熟站点组：Amazon US、CA、UK，外加 MX、BR 等新市场。
- Amazon 增长站点组：Amazon EU，以及全新平台 AU、JP。
- 北美增长平台组：Home Depot、Wayfair、Walmart，挖掘 US、CA 市场的平台机会。
- 每组 2-3 人配置，含 1 个 team lead。

本协议当前是经营管理框架，不是最终薪酬制度。具体奖金比例、基准值、财务口径和个人名单需要后续用真实经营数据校准。

## Core thesis

跨境电商运营绩效不应围绕单一销售额设计，而应围绕 `可控经营贡献` 设计。

绩效方案本质上是注意力分配系统：它决定团队会优先保护利润、增长、库存、账号健康、新品开发、平台开拓，还是只追求短期 GMV。

对 KES 当前团队，更合理的设计不是一套统一 KPI，而是按业务阶段拆成三类经营任务：

```text
成熟盘守利润和现金效率
增长盘买增长、孵新品、验证市场
新平台盘验证平台机会和组织能力
```

因此绩效方案应同时回答四个问题：
- 这个组的主要经营任务是什么？
- team lead 和组员分别能控制什么？
- 哪些指标用于奖励，哪些指标用于约束和扣分？
- 怎样避免短期冲量伤害库存、利润和平台资产？

## Business unit map

### 1. Amazon 成熟站点组

范围：
- Amazon US、CA、UK。
- 外加 MX、BR 等新市场，但 MX、BR 应先作为成熟组下的“外延市场实验”，不要完全按 US/CA/UK 的成熟盘逻辑考核。

经营定位：
- 核心任务是守住成熟利润池、现金效率、账号质量和主力 ASIN 的长期竞争力。
- 这组不应被单纯要求高速增长，否则容易通过降价、加广告、堆库存制造表面增长。
- 对 MX、BR 这类新市场，应在成熟组内设独立实验指标，避免被成熟站点利润指标压死。

主要绩效逻辑：
- 贡献利润和利润率是核心。
- 库存周转、断货率、滞销库存是强约束。
- 广告效率应看总利润和自然排名的关系，不应只压 ACOS。
- 新市场应看验证进度、有效 SKU 数、基础运营闭环，而不是早期绝对利润。

### 2. Amazon 增长站点组

范围：
- Amazon EU。
- 全新平台/站点：AU、JP。

经营定位：
- EU 在利润率较高的前提下，应承担更多新品开发、产品升级、价格带扩展和类目机会验证。
- AU、JP 属于更早期市场，应以建立平台认知、合规/本地化能力、首批可复制 SKU 为阶段目标。
- 这组的关键不是短期利润最大化，而是把高利润市场转化为可持续的产品和市场拓展引擎。

主要绩效逻辑：
- 成熟 SKU 看贡献利润和增长质量。
- 新品看阶段性里程碑，不宜过早用成熟利润考核。
- EU 要单独考虑 VAT、合规、语种、本地仓和国家差异，不能只看泛欧洲销售额。
- AU、JP 要设置探索期，重点看验证质量、上架完整度、早期转化、复盘闭环。

### 3. 北美增长平台组

范围：
- Home Depot。
- Wayfair。
- Walmart。
- 目标是挖掘 US、CA 市场中 Amazon 以外的平台机会。

经营定位：
- 这组不是 Amazon 运营的复制组，而是渠道开发和平台适配组。
- Wayfair 已有稳定业务收入，V2 文件显示 2026 年 1-3 月约 5.7-6.6 万美元/月，核心任务是提高收入质量、贡献利润、SKU 扩张效率和运营可复制性。
- Home Depot 当前约 2.5 万美元/月，阶段目标是 6 月稳定到 3 万美元/月，核心任务是解决核心 SKU 缺货、上新审核慢、PDP 内容转化和广告效率。
- Walmart 当前仍是每月几百美元收入的冷启动阶段，核心任务是跑通基础运营闭环、找出有效 SKU 和验证履约/内容/价格模型。
- Home Depot、Wayfair、Walmart 的用户心智、内容要求、履约和平台运营机制不同，应按平台建立机会档案。
- 三平台必须加入产品版本与价格分层：Amazon Core、Walmart Entry、Home Depot / Wayfair Premium，避免完全同款跨平台比价互相压价。

主要绩效逻辑：
- Wayfair / Home Depot：看收入增长、贡献利润、退货/破损、履约、内容转化、有效 SKU 扩张和平台运营 SOP。
- Walmart：看基础闭环、有效 SKU、Listing Quality、WFS/自发货、早期曝光/点击/订单和冷启动复盘。
- 三个平台不能用同一套探索期 KPI。

## Role split

每个组 2-3 人，含 1 个 team lead，绩效要区分组绩效与个人绩效。

### Team lead

team lead 的责任是经营单元负责人，而不是高级执行员。

应负责：
- 目标拆解：把年度/季度经营目标拆成站点、平台、SKU、项目。
- 资源配置：决定成熟品维护、新品孵化、广告预算、库存优先级。
- 经营复盘：解释利润、增长、库存和广告变化，而不是只报结果。
- 风险控制：账号健康、库存、负毛利、违规、关键 SKU 断货。
- 人员培养：让 2-3 人的小组形成可复制分工，而不是所有判断都集中在 lead 身上。

team lead 的绩效应更多绑定组经营结果和组织能力，而不是个人操作量。

### 组员

组员的责任应按可控工作包拆分：
- 站点/平台日常运营。
- Listing、A+、图片、内容更新和本地化。
- 广告执行和搜索词整理。
- 新品上架、竞品监控、review/Q&A 反馈。
- 库存预警、异常处理、活动提报。
- 平台资料、合规资料、后台运营流程沉淀。

组员的绩效应更多看过程质量、负责模块结果、协同响应和可复用输出。

## Indicator architecture

建议采用四层指标，而不是单一 KPI：

### 1. 经营结果指标

用于判断公司最终是否得到想要的结果。

优先级从高到低：
- 贡献利润。
- 毛利额。
- 销售额。
- 增长率。
- 回款/现金效率。
- 库存周转。

原则：
- 成熟业务尽量用贡献利润，而不是销售额。
- 增长业务可在阶段内允许利润率下降，但必须解释增长买来的是什么资产：新品、排名、评价、平台资质、渠道数据，还是只是亏损销量。

### 2. 过程指标

用于判断运营是否做了正确动作。

可选指标：
- Listing 更新和质量评分。
- A+ / 图片 / 视频 / 本地化完成度。
- 广告结构调整和 search term 复盘频率。
- 竞品监控和价格带变化记录。
- 新品上架节点完成率。
- 活动提报和复盘质量。
- review / Q&A / 差评问题归因。
- 平台资料和 SOP 沉淀。

原则：
- 过程指标不能过多，避免变成填表游戏。
- 每个过程指标都应能解释其对利润、转化、风险或组织能力的贡献。

### 3. 风险约束指标

用于防止短期行为。

建议作为门槛、扣分或奖金系数，而不是普通加分项：
- 账号违规。
- 负毛利订单比例。
- 断货率。
- 超龄库存和滞销库存。
- 广告失控。
- 退货率/取消率异常。
- 评分下降。
- 合规资料缺失。
- 平台 SLA 违规。

原则：
- 严重账号风险应直接触发奖金降档或归零。
- 因公司库存、供应链或资金决策导致的问题，不应全部归责运营个人，但运营必须提前预警并留痕。

### 4. 组织资产指标

用于奖励长期能力建设。

可选指标：
- 可复用站点 SOP。
- 平台规则和后台操作手册。
- 新品复盘模板。
- SKU 生命周期看板。
- 竞品和价格带数据库。
- 广告搜索词资产。
- 本地化内容规范。
- 平台机会档案。

这类指标尤其适合 team lead 和增长平台组。

## Recommended scorecards

### Amazon 成熟站点组 scorecard

建议权重：

| 模块 | 权重 | 说明 |
|---|---:|---|
| 贡献利润 / 毛利额 | 35%-45% | 主力利润池，避免只看 GMV |
| 销售目标与市场份额 | 15%-20% | 保持成熟站点基本增长和竞争位置 |
| 库存与现金效率 | 15%-20% | 周转、断货、滞销、补货准确性 |
| 广告与内容效率 | 10%-15% | 广告结构、自然排名、转化内容 |
| 风险与账号健康 | 10%-15% | 作为强约束，可设扣分/归零项 |
| 新市场实验 | 5%-10% | MX、BR 等按里程碑单独考核 |

适合的奖金逻辑：
- 基础奖金看目标完成度。
- 超额奖金基于超额贡献利润，而不是总销售额。
- MX、BR 不宜吞掉成熟站点权重，应单独设实验奖金或项目分。

### Amazon 增长站点组 scorecard

建议权重：

| 模块 | 权重 | 说明 |
|---|---:|---|
| 贡献利润与高利润市场增长 | 25%-35% | EU 的利润质量仍要被看见 |
| 新品开发与上新成功率 | 25%-35% | 核心任务是把利润市场变成产品引擎 |
| 市场/站点拓展里程碑 | 15%-20% | AU、JP 的探索、上架、验证、复盘 |
| 内容、本地化与转化效率 | 10%-15% | EU 语种、本地合规、页面表达 |
| 库存、合规与账号风险 | 10%-15% | EU/AU/JP 的合规和履约风险更高 |

适合的奖金逻辑：
- 对成熟 SKU 用利润目标。
- 对新品用阶段性里程碑：上架、有效流量、稳定出单、盈亏平衡、达到目标利润。
- 对 AU/JP 设探索期，先奖励有效验证和可复制 SOP，避免用短期利润否定早期市场。

### 北美增长平台组 scorecard

建议拆分为稳定收入平台和冷启动平台两套权重。

Wayfair / Home Depot 当前阶段建议权重：

| 模块 | 权重 | 说明 |
|---|---:|---|
| 收入与贡献利润 | 25%-35% | Wayfair 已稳定高于 5 万美元/月；Home Depot 当前要从 2.5 万美元/月稳到 3 万美元/月 |
| SKU 经营质量 | 15%-25% | top SKU 稳定性、新增 SKU 有效率、缺货和价格 |
| 履约与售后风险 | 15%-20% | 退货、破损、取消、replacement、客服 |
| 内容与平台转化效率 | 15%-20% | Wayfair 属性/场景内容，Home Depot 项目型内容和 retail media |
| 平台能力沉淀 | 5%-15% | SOP、资料包、复盘、可复制打法 |
| 产品/价格分层 | 5%-10% | 避免完全同款跨平台比价，形成 Core / Entry / Premium 版本策略 |

Walmart 当前阶段建议权重：

| 模块 | 权重 | 说明 |
|---|---:|---|
| 基础运营闭环 | 20%-30% | 账号、付款、退货、WFS/自发货、后台流程 |
| 有效 SKU 与 Listing Quality | 25%-35% | 从几百美元/月找到可放大的 SKU 和内容/价格/配送模型 |
| 早期订单质量 | 15%-20% | 曝光、点击、订单、取消、退货、tracking |
| 履约和毛利验证 | 10%-20% | WFS/自发货、费用、退货、基础利润模型 |
| Walmart SOP 沉淀 | 5%-15% | 平台规则、冷启动复盘、淘汰标准 |
| 产品/价格分层 | 5%-10% | 与 Amazon / WF / HD 形成配置差异，避免直接比价 |

适合的奖金逻辑：
- Wayfair / Home Depot 应进入“经营结果 + 利润质量 + 风险控制”的奖金逻辑。
- Walmart 仍用“冷启动里程碑 + 有效 SKU + 订单质量”的奖金逻辑，暂不宜用贡献利润主导。
- 对同一组内不同平台，必须按阶段分账，否则 Wayfair/Home Depot 的稳定收入会掩盖 Walmart 的冷启动问题。
- V3 将 V2 的细任务收敛为平台管理综合考核，方向正确；但必须保留两条硬约束：一是平台阶段分账，二是产品/价格分层，否则综合考核会掩盖核心风险。

## New product performance method

新品必须从成熟品绩效中拆出来。

建议按阶段考核：

1. `launch readiness`
   - 产品资料齐全。
   - Listing / A+ / 图片 / 视频 / 本地化完成。
   - 合规、包装、说明书、物流方案确认。

2. `traffic and signal`
   - 获得有效曝光和点击。
   - CTR、CVR、加购、关键词表现达到最低信号线。
   - review / Q&A / 客服反馈开始形成可分析样本。

3. `commercial proof`
   - 稳定出单。
   - 广告和自然流量结构可解释。
   - 退货、差评和履约问题在可接受范围。

4. `profitability`
   - 达到盈亏平衡。
   - 毛利和广告效率进入目标区间。
   - 补货节奏和库存风险可控。

5. `scale or kill`
   - 决定放量、迭代、转平台、清仓或淘汰。

关键原则：
- 新品早期不应用成熟利润指标直接考核。
- 每个新品都要有 kill criteria，否则绩效会鼓励无限续命。
- 新品奖金应奖励“有效验证”，不只奖励“成功故事”。

## Team lead vs group bonus

建议把奖金拆成三层：

```text
公司/事业部奖金池
组奖金池
个人绩效分配
```

### 组奖金池

组奖金池适合绑定：
- 组贡献利润。
- 组增长目标。
- 组新品/平台项目里程碑。
- 组风险约束。

这样可以减少组内互相甩锅，尤其是 2-3 人小组里，运营、广告、内容、库存动作高度交叉。

### 个人分配

个人分配适合看：
- 负责站点/平台/SKU 的实际结果。
- 负责模块的过程质量。
- 关键项目贡献。
- 复盘和 SOP 沉淀。
- 协同质量。

team lead 应有一部分奖金直接来自组结果，另一部分来自团队建设和关键风险控制。

## Anti-gaming rules

绩效方案必须提前写清哪些行为不奖励：

- 靠低价和高广告制造亏损销售额。
- 为完成当月目标而牺牲下月库存和排名。
- 上架大量低质量 SKU 堆数量。
- 只追 ACOS，导致有效增长机会被错杀。
- 只追利润率，导致新品和新平台永远无法启动。
- 把公司本来已有的自然流量全部算成个人新增贡献。
- 不预警库存、合规、账号问题，事后归因给外部部门。
- 为了短期订单承诺平台无法稳定履约的交付方式。

## Implementation sequence

### Phase 1: 统一财务口径

先定义每个站点/平台的贡献利润口径：

```text
销售收入
- 产品成本
- 平台佣金/交易费
- FBA/仓储/尾程/平台履约费
- 广告费用
- 退货/赔付/折扣
- 其他可归因直接费用
= 贡献利润
```

如果暂时无法精确到 SKU，先用站点/平台级别口径，但要标明数据缺口。

### Phase 2: 划分业务阶段

每个站点、平台、SKU 应标记阶段：
- mature：成熟利润池。
- growth：增长和放量。
- launch：新品/新站点启动。
- exploration：平台机会探索。
- harvest：清仓、收割、维护。
- kill：淘汰退出。

不同阶段使用不同指标权重。

### Phase 3: 先模拟，不立即强执行

建议用过去 3-6 个月数据模拟：
- 如果用该方案，会给出怎样的奖金结果？
- 是否奖励了真正赚钱和长期有价值的行为？
- 是否误伤了新品、新平台或不可控供应链问题？
- 是否让 team lead 有动力培养人和沉淀流程？

### Phase 4: 试运行 1-2 个周期

先做影子绩效或半权重执行：
- 月度看执行和风险。
- 季度看利润、增长、新品和平台验证。
- 每次复盘都记录指标失真点。

### Phase 5: 固化制度

固化时应写清：
- 指标定义。
- 数据来源。
- 计算周期。
- 目标制定方法。
- 奖金池和分配公式。
- 红线和扣分规则。
- 异常申诉机制。
- 谁有权调整目标。

## Open questions

后续需要补齐：
- 各组当前人员名单、具体岗位和可控边界。
- US、CA、UK、EU、MX、BR、AU、JP 各站点近 6-12 个月销售、毛利、广告、库存数据。
- Home Depot、Wayfair、Walmart 当前账号状态、SKU 数、订单量、平台限制和履约方式。
- Wayfair / Home Depot 近 6-12 个月收入、毛利、退货、破损、广告/活动、SKU 贡献分布。
- Walmart 当前上架 SKU、Listing Quality、曝光、点击、订单、WFS/自发货状态和阻塞项。
- Amazon / Walmart / Home Depot / Wayfair 同源产品的版本、配置、价格、毛利和比价风险。
- KES 对 2026 年的优先级：利润、增长、新品、清库存、新平台，哪个优先。
- 是否希望把 team lead 设计成小 P&L 负责人。
- 奖金预算边界：希望绩效奖金占固定薪酬的比例，以及超额利润分享上限。

## Review notes

当前判断：
- 成熟组应以贡献利润和现金效率为主。
- Amazon 增长组应承担新品和高利润市场扩张，不能用成熟盘规则压死探索。
- 北美增长平台组应按平台阶段拆分：Wayfair 已是高于 5 万美元/月的稳定平台，Home Depot 当前要从 2.5 万美元/月稳到 3 万美元/月，二者应考核收入质量、贡献利润和可复制增长；Walmart 仍是冷启动，应考核基础闭环、有效 SKU 和早期订单质量。
- 三平台绩效必须把产品/价格分层作为硬约束，防止同款跨平台比价导致利润系统性下移。
- 每组 2-3 人的小团队，适合“组奖金池 + 个人贡献分配”，否则容易把高度协同的结果错误归因到个人。
