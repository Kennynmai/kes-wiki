# 产品开发洞察采集模板

## 用途
用于把产品开发过程中的原创判断、失败教训、测试设计、实验结果、材料取舍、结构权衡，先采集成结构化记录，再决定是否转成：
- 官网 `engineering note`
- 官网 `test note`
- 官网 `how we decided` 页面
- 站外长文 / 视频脚本 / 专家型分发内容

- 项目 / 产品：
- 记录日期：
- 记录人：
- 所属阶段：
  - 概念阶段
  - 打样阶段
  - 测试阶段
  - 改版阶段
  - 上市前阶段
  - 上市后优化阶段
- 关联文件 / 群聊 / 测试记录：

## 二、这次最值得保留的原创洞察
- 一句话洞察：
- 这条洞察解决了什么问题：
- 这条洞察最适合服务哪个内容目标：
  - 建立专业感
  - 解释为什么这样设计
  - 回答常见质疑
  - 支撑对比内容
  - 支撑安装 / 使用 / 维护内容
  - 支撑 AI 可引用答案

  - 供应链限制

  - `测试后我们发现什么`

  - `选购判断指南`

  - 竞品拆解
