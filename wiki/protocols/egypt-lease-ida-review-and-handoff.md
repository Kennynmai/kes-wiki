# 埃及工厂租赁与 IDA 申请阶段 Review 与信息交代清单

## Trigger / scope
- 适用于当前阶段：
  - 埃及本地公司已注册；
  - 银行账户已开立；
  - 即将签署工厂租赁合同；
  - 即将申请 IDA 经营许可；
  - 当地已有代表在推进。
- 目标不是重复论证“值不值得做”，而是让整个团队在签租约和提交 IDA 前，对关键风险、资料链和授权边界形成共同认识。

## Inputs
- 公司注册资料包：章程、商业登记、税卡、银行开户资料、授权文件。
- 厂房资料包：房东身份、产权/出租权、现有许可、图纸、公用事业账单、消防资料。
- IDA 资料包：拟申请活动、产品、面积、设备清单、文件进度。
- 代表推进资料：谁在对接、谁签字、谁能提交、谁能付款、谁有印章和网银权限。
- 商业资料：产品范围、预计设备、客户类型、预计投产节奏。

## Steps

### 1. 先让团队统一当前阶段判断
- 现在项目已经从“前期研究”进入“执行前控制”阶段。
- 当前最大的风险不是公司设立，而是四个执行失误：
  - 签了一个后续无法支持 IDA 的租约；
  - 提交了材料不完整或逻辑不一致的 IDA 申请；
  - 当地代表在权限不清的情况下做了不可逆承诺；
  - 团队以为“证在申请中”就等于“可以先做装修、先上设备、甚至先开工”。
- 团队必须明确：`租赁合同`、`Sond Hiyaza / سند حيازة`、`IDA 申请口径`、`授权链` 是一体的，不是四个独立事项。

### 2. Review findings

#### [P1] 最大的即时风险是租赁合同与 IDA 的 `سند حيازة` 要求不匹配
- IDA 官网的经营许可页面把 `سند حيازة المنشأة` 列为核心材料。
- IDA 2019 年《الدليل الارشادي لسندات الحيازة》进一步说明：租赁作为占有依据时，不仅要有租约，还要求租约经过 `إثبات تاريخ / محضر تصديق`，并附房东产权或其对标的物权利依据，以及房东关于无建筑违法、无争议等声明。
- 这意味着：如果租约文本、签约主体、厂房地址、面积、用途、房东权利链、日期固定方式之间有任何断裂，后面可能不是“补一份材料”，而是整个许可基础文件失效。

#### [P1] 第二个高风险点是租约先签了，但拟申请活动与厂房当前许可状态不匹配
- IDA 的工业活动查询是按具体工业活动决定许可路径，而不是按“行业大类”粗放处理。
- 如果你们实际要做的是 `裁剪 + 缝制 + 包装 + 成品仓储 + 安装配套`，就不能只用“纺织相关”或“轻工业厂房”这种口径去签约和申报。
- 一旦活动口径、设备口径和厂房既有许可状态不一致，最常见后果不是直接被拒，而是进入补材料、整改、改申请、改合同的连锁拖延。

#### [P1] 第三个高风险点是当地代表成为“单点失控”
- 当地有代表推进本身是优势，但只要以下边界不清，就会变成控制风险：
  - 代表是否有签约权；
  - 是否能代表公司作承诺；
  - 是否能控制原件、印章、网银、UKey、密码；
  - 是否能自行选律所、顾问、厂房；
  - 是否需要双重批准才能提交政府材料或付款。
- 项目进入租赁和许可阶段后，任何“先推进再补批”的习惯都容易制造既成事实。

#### [P1] 第四个高风险点是团队把 IDA 申请当成“纯跑流程”，低估技术文件链
- IDA 官网列出的材料不只是公司证照，还包括：
  - 技术与环保文件；
  - 火灾风险防范报告或既有 civil defense approval；
  - 经认可工程师出具的建筑可占用/适用证明。
- 这意味着 IDA 并不只是法务和关系推进问题，而是 `法务 + 工程 + HSE + 文件逻辑一致性` 的联合动作。
- 如果团队内部没有明确一个人统筹“最终提交版本”，很容易出现不同顾问给不同口径的材料。

#### [P1] 不要把“800 平方米以下消防要求降低”当作已确认规则
- 本次检索未找到 IDA、Civil Defense 或 Egyptian Fire Code 的官方公开依据，能支持“工业厂房 800 平方米以下消防要求降低”这个具体说法。
- IDA FAQ 的官方口径是：Civil Defense approval 是工业许可的基本组成部分；通知制 license 的差别只是可以先取得许可、再推进消防民防程序，并在检查后给整改期。
- IDA FAQ 还明确：如果检查发现问题，投资人可被要求到 IDA 说明，随后获得最多 180 天整改期；小型和微型项目可获得加倍整改期。这是“整改期限/程序便利”，不是免除或降低实体消防要求。
- 因此，任何中介或律师如果提出 800 平方米阈值，必须要求其提供具体阿文法规、条文、适用建筑类型、适用活动风险等级、以及 Civil Defense 书面确认。

#### [P2] 第五个风险是租约文本没把“工业使用”和“改造权”写死
- 对你们来说，可接受的租约不只是能办公或仓储，而是必须允许：
  - 指定工业活动；
  - 电力增容；
  - 消防整改；
  - 排风、照明、仓储和机电改造；
  - 设备进场、安装、运行和拆除搬离。
- 如果这些权利写得模糊，后续即使拿到证，也可能在施工或设备安装时被房东卡住。

#### [P2] 第六个风险是“资料已交给代表”但团队没有一套总台账
- 进入政府申报和合同阶段后，最怕的是文件散在微信、邮箱、扫描件和本地代表手里。
- 没有总台账时，你们很难确认：
  - 哪些是原件；
  - 哪些是签字版；
  - 哪些已经提交政府；
  - 哪些版本还在修改；
  - 哪些材料的名称、地址、面积、法人、活动描述彼此不一致。

#### [P2] 第七个风险是开工边界不清
- 团队需要在内部提前定一个红线：
  - 什么叫“可以签租约”；
  - 什么叫“可以付押金/首期租金”；
  - 什么叫“可以做装修和机电改造”；
  - 什么叫“可以订设备和发运设备”；
  - 什么叫“可以正式生产”。
- 如果不定义这些节点，当地执行通常会自然前冲，形成时间压力，逼总部追认。

## Failure modes
- 租约地址和拟申报地址写法不一致。
- 房东不是产权人，也没有完整出租授权链。
- 租约签了，但未做日期固定/认证，不满足 `سند حيازة` 实操要求。
- 公司名称、商业登记名称、租约承租人名称、IDA 申请主体名称不一致。
- 申请口径写“纺织”，实际设备和流程涉及更具体活动，后续被要求技术变更。
- 技术/环保/消防材料由不同顾问分别出具，版本不一致。
- 当地代表掌握印章或网银，但缺乏双签或复核机制。
- 为了赶进度先付长期租金，后续却因证件或改造问题无法投产。

## 信息交代要求清单

### A. 对全团队必须讲清的“现状事实”
- 公司已注册完成到什么层级：
  - Commercial Register
  - Tax Card
  - Bank account
  - Chamber / federation membership
  - 是否已开通网银和谁控制
- 当前还没完成什么：
  - 工厂租赁合同未签
  - IDA 经营许可未获批
  - 是否尚未取得 civil defense / fire 文件
  - 是否尚未完成技术环保文件
- 哪些动作不能默认已经具备合法性：
  - 装修
  - 安装生产设备
  - 试产
  - 正式生产

### B. 对租赁合同必须收齐并解释给团队的文件
- 房东身份证明 / 公司主体文件。
- 产权证或其他合法占有和出租依据。
- 如房东非产权人，完整授权链。
- 厂房准确地址、面积、楼层、附属仓库、装卸区。
- 当前许可/用途文件。
- 电力容量、历史电费/水费账单。
- 消防资料、既有整改记录。
- 租约签字版、盖章版、阿拉伯文正本与工作翻译版。
- 日期固定/公证/认证路径说明。

### C. 对 IDA 申请必须收齐并解释给团队的文件
- 拟申请活动名称与代码。
- 申请是 `إخطار` 还是 `مسبق`，依据是什么。
- 是否考虑通过 `مكاتب الاعتماد` 路径推进。
- IDA 官方材料清单的逐项状态：
  - recent Commercial Register
  - POA / bank authorization
  - `سند حيازة`
  - federation membership
  - technical & environmental file
  - fire-prevention report / prior civil defense approval
  - tax card
  - engineer occupancy / suitability certificate
  - company contact statement
- 每份材料的：
  - 提供方
  - 当前状态
  - 原件/扫描件
  - 最后版本日期
  - 是否已提交

### D. 对当地代表必须书面澄清的授权边界
- 代表是否能签租约。
- 代表是否能代表公司承诺租期、押金、违约责任、装修责任。
- 代表是否能付款；付款限额是多少。
- 代表是否能选择或更换律师、顾问、中介。
- 代表是否保管：
  - 公司印章
  - 银行 UKey / token
  - 网银密码
  - 原始证照
- 哪些事项必须先经香港/总部书面批准。

### E. 对管理层必须呈报的一页状态板
- 当前阶段：`Lease pending / IDA pending`
- 当前最高优先事项：只列 3 项。
- 当前最大阻塞：只列 3 项。
- 下一个不可逆动作是什么。
- 如果今天停止推进，沉没成本是多少。
- 如果今天强行推进，最大的合规风险是什么。

## Required document tracker
- 建议立刻建立一份单表，至少包含以下列：
  - Document name
  - Arabic / English / Chinese name
  - Issuing party
  - Legal entity shown
  - Site address shown
  - Area shown
  - Activity shown
  - Signature / chop status
  - Original held by
  - Soft copy link
  - Last updated date
  - Submitted to / not submitted
  - Notes / mismatch

## Go / no-go gates

### Gate 1: Can sign lease
- 房东权属链清楚。
- 租约主体、地址、面积、用途准确。
- 已明确 `سند حيازة` 的日期固定/认证路径。
- 租约明确工业使用、改造权、设备安装和搬离权。
- 付款条件与交付条件匹配，不是先大额付款后补条件。

### Gate 2: Can submit IDA
- 活动口径已锁定。
- 租赁文件链完整。
- 技术环保文件、消防材料、工程师证明准备完成。
- 公司与厂房信息在所有材料中一致。
- 提交版本由一名负责人做最终一致性审查。

### Gate 3: Can commit capex
- 厂房技术核查通过。
- 租约和许可证路径已稳定。
- 设备清单与核定活动匹配。
- 没有未解决的地址、面积、用途或消防矛盾。

## Review notes
- IDA 官网当前页面显示：
  - `إصدار رخصة التشغيل بنظام الإخطار` 的法定时长为 1-5 个工作日；
  - `إصدار رخصة التشغيل بنظام المسبق` 的法定时长为 30 天；
  - 通过 `مكاتب الاعتماد` 可走 same-day 提交路径，但前提是材料完整且取得 accredited office certificate。
- 这不应被理解为“关系够强就能快速过”，而应理解为“只要材料链不完整，时钟就不会按理想速度走”。
- 当前阶段最值得追加投入的不是更多口头跟进，而是：
  - 一名强法务/牌照负责人；
  - 一名文件总控；
  - 一次厂房技术核查；
  - 一套明确的代表授权矩阵。
