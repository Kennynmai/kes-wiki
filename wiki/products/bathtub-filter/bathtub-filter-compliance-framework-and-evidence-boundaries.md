# 浴缸过滤器（bathtub filter）合规框架与证据边界

## 为什么有这页
这页不是列品牌，而是建立一个 **判断框架**：

当一个 bathtub filter 页面写到：
- tested
- certified
- NSF / ANSI
- clinically tested
- uses certified media
- removes contaminants

我们应该怎么拆解，才不会被 marketing language 误导。

## 一套最重要的拆解顺序
### 第 1 层：到底在说什么对象？
先分清楚页面说的是：
- **finished product（整机 / 成品）**
- **cartridge / filter media（滤芯 / 介质）**
- **brand lineage（品牌既有技术血统）**
- **single test condition（单一测试条件）**

### 第 2 层：到底是什么性质？
再分清楚它说的是：
- **official certification**
- **third-party testing**
- **internal testing**
- **media credibility only**
- **testimonial / expert endorsement**

### 第 3 层：到底覆盖什么？
最后再问：
- 覆盖的是 chlorine only，还是 broad contaminant stack？
- 覆盖的是 specific condition，还是 everyday use？
- 覆盖的是 aesthetic / comfort issue，还是 health / skin outcome？

## bathtub filter 里最常见的五种误读
### 1. 把 `tested to standard` 读成 `certified to standard`
这是最常见、也最危险的混读之一。

### 2. 把 `uses certified media` 读成 `whole product certified`
介质可信，不等于整机经过同等级验证。

### 3. 把 `chlorine reduction standard` 外推成 `broad contaminant removal`
标准常常是窄覆盖，不等于什么都能讲。

### 4. 把 `clinical wording` 读成强结果证明
`clinically tested`、`clinically proven`、`doctor recommended` 很容易形成权威感，
但它们的证据强度、测试边界和法律含义差异极大。

### 5. 把 `testimonial` 当作 efficacy evidence
用户或 influencer 的使用感受，不等于 finished-product proof。

## KES 可直接使用的审核问题
当以后看任何 bathtub filter 页面时，可以直接连问这 8 个问题：
1. 这是在说整机、滤芯、介质，还是品牌血统？
2. 这是 certified、tested，还是仅仅提到了某个标准？
3. 有没有 public listing / certificate / report 可核验？
4. 覆盖的是 chlorine，还是宽泛污染物？
5. 测试条件是否接近日常 tub-fill conditions？
6. 有没有把 media credibility 暗示成 whole-product efficacy？
7. 有没有把 testimonial / expert quote 暗示成医学证明？
8. 有没有把 comfort framing 偷换成 eczema / baby-safety implication？

## 当前对 KES 的实务建议
### 更适合采用的表达纪律
- 先定义 object（整机 / 滤芯 / 介质）
- 再定义 evidence type（certified / tested / internal / testimonial）
- 最后定义 scope（只针对 chlorine？还是别的？）
- 对任何 badge / seal / authority name，都补一行 basis 说明：**certifier、standard、claim scope、public proof**

### 更适合避免的表达纪律
- 用一个标准编号暗示整包健康结果
- 用一个 tested 词汇暗示正式认证
- 用 certified media 暗示 certified product
- 用 expert / doctor 语言抬高页面可信度，却不给可核验边界
- 用 logo impression 代替具体 scope

## 可直接套用的页面审校模板
| 字段 | 必填问题 | 通过标准 |
|---|---|---|
| issuer | 谁发的？ | 具体 certifier / lab / supplier 名称 |
| object | 证据对应整机、滤芯还是 media？ | 对象唯一，不跳变 |
| scope | 到底支持 chlorine、materials 还是 comfort？ | 不跨越证据边界 |
| proof | 能公开查什么？ | listing / report / none 说清 |
| wording | 页面最终写什么？ | 不把 tested 写成 certified |

## 对 bathtub filter 这题的总结
在这个品类里，真正的竞争力不只是“谁更会写文案”，而是：

> **谁能把证据边界写清楚，还不把自己写得像弱产品。**

这也是为什么 KES 如果进入，最值得坚持的路线仍然是：
- narrow claim surface
- clear test boundary
- honest fit scope
- premium but disciplined expression
